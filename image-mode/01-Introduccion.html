Introducción a RHEL Image Mode

Bootable Containers

Bootable containers o bootc es una herramienta que se utiliza para crear contenedores que incluyen el kernel, el cargador de arranque u otros componentes que no se incluyen en los contenedores normalmente. Una vez creado un contenedor de este tipo, se sube a un registry, al igual que cualquier contenedor y mediante image builder podemos crear imágnenes de RHEL que se desplegarán sobre cualquier infraestructura, máquinas físicas, virtuales o en proveedores cloud.

Esta nueva forma de gestonar el sistema operativo también permite actualizar dichos componentes a partir de imágenes de contenedores bootables.

Setup Kubernetes

The profile my_profile is created to run the tutorial:

  • Minikube

  • OpenShift

Having minikube installed and in your PATH, then run:

MacOS

minikube start --memory=8192 --cpus=3 --kubernetes-version={kubernetes-version} --vm-driver=virtualbox -p my_profile

Linux

minikube start --memory=8192 --cpus=3 --kubernetes-version={kubernetes-version} --vm-driver=kvm2 -p my_profile

Windows:

minikube start --memory=8192 --cpus=3 --kubernetes-version={kubernetes-version} --vm-driver=hyperv -p my_profile

And the output must be something similar like:

Finally configure to use minikube internal docker as docker host:

eval $(minikube docker-env -p my_profile)

To run OpenShift4, you need to have one provisioned using try.openshift.com or can use any existing OpenShift4 cluster. Once you have your cluster, you can download the latest OpenShift client(oc) from here and add to your path.

oc version

You can check the OpenShift version using:

oc version

The output should show oc version >=4.7:

Client Version: 4.7.0-202102130115.p0-c66c03f
Kubernetes Version: {kubernetes-version}

Then login into the OpenShift cluster using oc login

And then you are ready for start using Kubernetes:

Get tutorial sources

Before we start setting up the environment, let’s clone the tutorial sources and set the TUTORIAL_HOME environment variable to point to the root directory of the tutorial:

git clone https://github.com/redhat-developer-demos/rhd-tutorial-common.git my_folder
export TUTORIAL_HOME="$(pwd)/my_folder"
cd $TUTORIAL_HOME
